Issues Guide


How Issues Work in Archa


We use GitHub Issues to track everything.

Types of Issues


* ``bug`` - Something is wrong with license text or site
* ``docs`` - Documentation improvement
* ``question`` - Clarification needed
* ``proposal`` - Change to Archa v1.1
* ``sponsor`` - Commercial / Threshold Entity inquiry

Before Opening an Issue


1. Search existing issues: https://github.com/newssungoldentoday-dev/archalics/issues
2. Check docs: https://newssungoldentoday-dev.github.io/archalics/
3. Check :doc:`problem` and :doc:`solution`

Opening a Good Issue

.. code-block:: text

   Good: "Section 3.2 unclear for non-profits with < $5M revenue"
   Bad: "License doesn't work"

Include:

* Which section (e.g., 3.2, 3.3)
* Your use case
* Suggested wording if you have one

Issue Lifecycle

Open -> Triaged -> Discussion -> Accepted / Rejected -> Closed
